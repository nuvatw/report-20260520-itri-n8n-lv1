# 首頁瀏覽體驗標準

第三週目標是讓首頁在報告數量增加後仍然好找、好掃描、好回溯。首頁需同時支援搜尋、分類、年度、月份、最新報告標示與清楚的空狀態。

## 必備互動

- 搜尋：可搜尋標題、副標、分類、客戶、期間、timeline 與檔案路徑。
- 分類：顯示全部分類，並在每個分類旁顯示報告數量。
- 年度：顯示全部年度與各年度報告數量。
- 月份：依目前年度與分類條件顯示可用月份。
- 清除條件：有任一條件啟用時，側欄顯示條件 token 與清除按鈕。
- 最新標示：日期最新的報告卡片需標出 `Latest`。
- 空狀態：無索引、無報告、無符合條件時需顯示不同訊息。

## 呈現規則

- 預設依日期由新到舊排序。
- month group 使用 `YYYY.MM`。
- category chip、year chip、month chip 都需保留 `aria-pressed`。
- 報告卡片需保留可點擊整卡連結。
- 行動版不得讓「開啟報告」換行。

## 第三週驗收清單

- 首頁存在 `#year-chips`。
- 首頁存在 `#month-nav`。
- 首頁存在 `#active-filters` 與 `#clear-filters`。
- 首頁存在 `#month-count`。
- 首頁產生 `.latest-badge`。
- 搜尋、分類、年度、月份四種 state 都參與 `filteredReports()`。
- 空狀態可提供清除條件行動。
- `npm run verify` 通過。
