# 擴充決策矩陣

第五週目標是評估下一階段擴充方向，避免首頁在報告增加後變成臨時功能堆疊。

## 決策摘要

| 擴充項目 | 建議狀態 | 原因 | 啟動條件 |
| --- | --- | --- | --- |
| 縮圖 | 暫緩 | 目前報告多為 A4 HTML，首頁文字資訊已足夠；縮圖會增加素材維護成本。 | 報告超過 12 份，且多數報告有穩定封面圖。 |
| PDF 下載 | 可選欄位先行 | 很多成果書後續會需要轉寄或存檔，先保留 metadata 欄位。 | 每份 HTML 都有對應 PDF，或客戶要求下載版。 |
| 客戶分類 | 已先行 | 客戶/合作單位會成為未來搜尋與分眾入口。 | 報告超過 8 份後，評估是否做首頁 clientGroup chips。 |
| 權限分流 | 暫緩 | 純靜態網站無法提供真正權限；目前僅提供 `visibility` 標籤。 | 需要登入、非公開客戶內容或內外部雙版本。 |
| 自動部署 | 建議下一階段 | 第四週已具備 release check，可接 CI/CD。 | 有 Git repo 與固定部署平台後啟動。 |
| PDF 生成 | 暫緩 | 報告 HTML 目前以螢幕與列印友善為主；自動 PDF 需另做版面測試。 | 客戶交付以 PDF 為主要格式。 |

## Metadata 擴充欄位

第五週已在 schema 與索引器保留以下選填欄位：

```json
{
  "clientGroup": "政府與公共部門",
  "tags": ["AI 素養", "青年政策"],
  "thumbnail": "assets/thumbnails/example.png",
  "pdf": "downloads/example.pdf",
  "visibility": "public"
}
```

規則：

- `clientGroup`：給未來客戶分類篩選使用。
- `tags`：納入搜尋索引。
- `thumbnail`：目前只保留欄位，首頁尚不顯示。
- `pdf`：目前只保留欄位，首頁尚不顯示。
- `visibility`：只作為標籤，不等於權限控管。

## 下一階段建議

優先順序：

1. 自動部署：把 `npm run release:check` 接到 GitHub Actions 或部署平台。
2. 客戶分類 UI：當報告數超過 8 份後加入 clientGroup 篩選。
3. PDF 下載：當每份報告都有 PDF 版本時加入首頁下載按鈕。
4. 縮圖：當封面素材穩定後再加入，不建議現在先做。
5. 權限分流：若有非公開內容，改採有登入機制的平台或分站部署。
