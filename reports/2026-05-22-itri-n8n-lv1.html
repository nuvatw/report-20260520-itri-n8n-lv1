報告摘要

Executive Summary

本報告目的有二：完整呈現本次十小時課程的學習成果，並將課程內容轉化為可供後續複習、內部溝通與流程落地的參考文件。

生成式 AI 的價值不只在單點內容生成，而在於帶動知識型工作的生產力提升，同時逼迫組織重新思考流程、角色與能力配置。McKinsey 指出，生成式 AI 可望對整體勞動生產力帶來顯著增益，但前提是組織願意投資於新技能與流程重整；NIST 的 AI 風險管理框架則提醒，生成式 AI 的應用往往需要額外的人工覆核、追蹤、文件化與管理監督。

本次課程的真正價值，在於它同時處理了「效率」與「治理」兩件事：一方面教會學員把資料流、節點、欄位與 AI 串接成自動化流程；另一方面也保留人工校稿、主管確認與資料留痕的思維。

02.0COURSE DESIGN
02

課程設計與教學架構

Course Design & Structure

本次教學採兩段式設計：先建立共通語言，再用專案反覆練習。

DAY 01 — FOUNDATION
6 小時

拉水 · 拉電 · 砌磚

建立資料流、節點思維與欄位結構化能力，奠定全課程的共通語言基礎。

DAY 02 — PRACTICE
4 小時

六個 Project 實作

透過專案反覆練習，把第一天的基礎能力轉成可落地的自動化流程。

METAPHOR 01
拉水

教資料怎麼流。讓學員理解 Fixed 與 Expression 的差異——Expressions 以 {{ }} 在節點參數內動態帶值，Edit Fields 則支援兩者切換。

METAPHOR 02
拉電

事件觸發與行動執行的流程觀念。n8n 官方將核心節點區分為 Trigger 與 Action，這正是自動化流程最根本的骨架。

METAPHOR 03
砌磚

把非結構化訊息轉成結構化欄位。先讓 AI 將文字整理成 JSON，再透過 Edit Fields 展開成可被資料表、試算表使用的多個欄位。

OpenAI 文件強調，把高優先級規則與使用者輸入分開，可讓模型更穩定遵循要求。若目標是把 AI 內容直接接入資料流，單純要求「輸出 JSON」並不保證符合欄位 Schema；要提升流程可靠度，Structured Outputs 才能在 Strict 模式下穩定要求模型輸出符合指定 JSON Schema 的資料。

03
03.0SIX PROJECTS
03

六個專案的學習成果

Six Projects — Learning Outcomes

六個專案從最小流程逐步堆疊到完整閉環，覆蓋了 n8n 最具可遷移性的核心模組。

PROJECT
01
練習內容

建立 Trigger 與 Action，將 Fixed 資料寫入 n8n 內建 Data Table。

帶走的能力

能做出最小可用流程，理解入口、動作、存檔三件事如何串起來。

PROJECT
02
練習內容

使用 Expression 取得動態時間，寫入資料表。

帶走的能力

理解靜態值與動態值差異，開始具備「流程會隨情境變化」的思維。

PROJECT
03
練習內容

建立超過一個 Action：寄送 Email，並把收件人與內容存回資料表。

帶走的能力

學會多動作鏈與留痕思維，不再只做單一步驟自動化。

從 n8n 官方能力對照來看，這些專案完整覆蓋了最具可遷移性的核心模組：Data Table 可直接儲存與管理結構化資料；Send Email 可寄送郵件並支援等待回覆；Loop Over Items 可批次處理多筆資料；Webhook 可承接外部事件；Schedule Trigger 可做定時催辦。後續頁面將接續 Project 04 至 Project 06 的成果說明。

04
03.1SIX PROJECTS
03

六個專案的學習成果（續）

PROJECT
04
練習內容

串接 OpenAI API 生成勵志內容，寄出並存下主旨與內容，且每次輸出不同。

帶走的能力

學會把 AI 納入內容生成流程，並讓內容產出與資料紀錄同步發生。

PROJECT
05
練習內容

用 Edit Field 建立變數，讓 AI 判斷三明治口味為鹹或甜並生成報告寄送；同時區分 User Message 與 System Message。

帶走的能力

學會把規則與輸入分層，讓 AI 不只寫內容，也能做分類、判斷與回報。

FINAL BOSS
06
練習內容

以 Loop 處理 20 筆資料，讓 AI 擔任 Cleaner、Decider、Producer——清洗姓名與 Email、判斷路徑、生成中文五言絕句或英文 Poetry、寄出並存庫，最後完成提醒。

帶走的能力

學會批次處理、多筆資料迴圈、AI 決策分流、資料清洗、內容產生、通知與資料留痕的完整鏈條。

Why Project 06 Matters

第六個專案讓學員真正進入工研院多數業務會遇到的真實水平：資料不只一筆、內容不只一種、判斷不只是與否、輸出不只是寄出。必須在清洗、判斷、分流、生成、寄送、存檔與完工通知之間完成閉環 —— 這正是從「會操作工具」走向「能設計流程」的分水嶺。

Next Step — Recommendation

最值得升級的地方是「結構化輸出標準化」。課堂已教會學員把資訊轉成 JSON 並拆成多欄位；下一步建議在正式進資料庫、Google Sheets、Notion 或名單系統的流程上，逐步導入 JSON Schema 與 Structured Outputs，讓 AI 不只「大致輸出正確格式」，而是穩定輸出符合欄位規格的資料，降低欄位遺漏、型別錯誤與後續補件成本。

05.0OVERALL EVALUATION
05

整體成效與結案評估

Overall Effectiveness

綜合課程內容、現場完成率與需求流程對照，本案可歸納出三項實質成果。

01

建立了共通語言

學員已能用 Fixed、Expression、Trigger、Action、JSON、Edit Field、Loop、清洗、分流、寄送、存檔等語言描述流程。團隊未來討論自動化時，已能直接進入「入口是什麼、欄位怎麼拆、哪裡要判斷、誰來核稿、資料存哪裡」的實作層次。

02

從單點操作到流程思維

拉水、拉電、砌磚不只是入門比喻，而是有效的拆解法：先學資料流，再學事件與動作，再學結構化。第二天以六個專案反覆驗證，使學員逐漸理解任何流程都能被拆成相似模組。

03

良好的學習轉移性

僅十小時總時數下，超過八成學員能完成所有 Project，說明難度控制得宜、教學順序符合成人學習節奏。多數學員已具備回到工作現場後「能與同事共同重建流程」的基礎，而非只能在課堂跟做一次。

結語與參考依據

Conclusion & References

本次課程的成果是——工研院相關團隊已在十小時內建立一套 AI 自動化的共通基礎語言，並能把這套語言對接到自身最迫切的六類工作情境。

在生成式 AI 持續推動知識型工作重組之下，課程價值在於讓組織提早完成基礎能力布建；治理面上，課程亦已把人工校稿、主管確認、資料留痕與批次追蹤的觀念帶進來，與 NIST 所強調的人機協作、角色責任與文件化管理方向一致。若後續能把專案進一步沉澱為可重複套用的模板、欄位規格與標準流程，本次十小時訓練的效益，將真正轉化為工研院後續流程升級的起點。
